# The Cookbook

Your wins, shared with the next student.

---

## What This Is

The prompt pack in `02-prompt-pack/` is the starter set. The cookbook is the *expansion pack* — built by students, for students.

If you've taken a prompt from this repo, modified it for a specific class, and made it work — submit it here. Future students will use what you built.

This is also the only way the repo stays alive long-term. **Two semesters from now, the most useful prompts in here will be the ones you contributed, not the ones the starter pack shipped with.**

---

## What Belongs in the Cookbook

✅ **Yes:**
- A modified version of an existing prompt that worked better for a specific class
- A brand-new prompt for a use case the starter pack doesn't cover (lab-report builder, debate-prep, internship interview prep, club-meeting agenda generator, etc.)
- A "prompt that almost worked but here's what I learned" post-mortem
- A side-by-side showing how a constraint change made the output dramatically better

❌ **No:**
- Anything containing real classmate names, professor names without permission, real essays you submitted, or anything that violates academic-integrity policy
- Prompts that produce essays-on-demand for graded assignments — that crosses the line from "use AI well" to "let AI replace your thinking"
- Anything containing API keys, personal info, or proprietary class material

## How to Submit

1. Fork the repo: [github.com/DevontiaW/brief-the-bot](https://github.com/DevontiaW/brief-the-bot) → "Fork" button.
2. In your fork, copy the [`_template.md`](./_template.md) file in this folder.
3. Rename your copy something specific — e.g., `lab-report-builder.md`, `debate-prep-stoss-side.md`, `internship-thank-you-email.md`.
4. Fill in the template (it's quick — see structure below).
5. Open a Pull Request back to the main repo.

Use the PR title format: `cookbook: <your prompt name>` — e.g., `cookbook: lab-report-builder`.

I review PRs weekly. Most get merged. If something needs revision I'll comment with a specific change.

## Template Structure

Every cookbook entry follows the same shape so they're easy to scan:

```markdown
# [Your Prompt Name]

[1-paragraph what-it-does]

## The Prompt
[the actual prompt, in a code block]

## What You Get Back
[example output snippet]

## When It Goes Wrong
[2–3 troubleshooting moves]

## Why I Built This
[1-paragraph: what class / what problem / what didn't work in the starter pack]

## Credit
**Built by:** [Your Name], [Your School / Class Year]
[Optional: GitHub handle, LinkedIn]
```

That's it. Copy [`_template.md`](./_template.md) and fill it in. ~30 minutes of work; lifetime contribution.

## What "Good" Looks Like

I'll prioritize merges that hit these marks:

1. **Specific use case.** "Prompt for biology lab reports" beats "general study prompt." The narrower the better — that's what makes the prompt actually work.
2. **Pulls all three levers.** ASK, CONTEXT, CONSTRAINTS visible in the prompt. Implicit doesn't count.
3. **Has constraints with teeth.** Numbered lists. Specific formats. Hard caps. Not vague advice.
4. **Honest about failure.** A "When It Goes Wrong" section that lists real failure modes is more useful than one that pretends everything worked perfectly.
5. **Voice is yours.** Don't try to match my voice. The cookbook is more interesting when every entry sounds like a different student.

## Recognition

Every merged PR puts your name on the cookbook entry. If you submit three or more accepted entries, you get listed in the repo's [`CONTRIBUTORS.md`](../CONTRIBUTORS.md) as a maintainer-track contributor.

I share notable prompts on LinkedIn (with credit + tag) — that's a portfolio piece for you. AI-fluent students who can ship are who recruiters want. This is one way to show it.
